How the load balancer picks an account
  1. Force header — If the request includes x-account-id, that account is used directly.
  2. Auto-fallback recovery — If a higher-priority account just recovered from a rate limit and Fallback is on, it gets promoted to the front of the line so requests shift back to it automatically.
  3. Session affinity — OAuth accounts have 5-hour conversation windows. If an account has an active session, keep using it to avoid burning a new window elsewhere.
  4. Priority + usage-aware selection — Among remaining candidates, pick by priority tier (lower = higher). Within the same tier:
    • Accounts under their reserve threshold come first.
    • Sorted by soonest reset time, so capacity about to expire gets used instead of wasted.
    • Soft reserve (default): account is deprioritized but still available as a last resort.
    • Hard reserve: account is fully excluded when above the threshold.
  5. Round-robin — Pay-as-you-go (API key) accounts at the same priority are rotated evenly.

Paused and rate-limited accounts are always skipped. The Next badge shows which account will handle the next request.
